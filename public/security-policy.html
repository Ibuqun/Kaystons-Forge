Vulnerability Disclosure Policy

Last updated: March 1, 2026

Kayston's Forge is a privacy-first, browser-based developer utilities suite. We take security seriously and welcome reports from the security community.

Scope

This policy covers the Kayston's Forge web application at kaystons-forge.vercel.app and its open-source repository at github.com/Ibuqun/Kaystons-Forge.

In Scope

  • Cross-site scripting (XSS) in tool previews or output rendering
  • Content Security Policy bypass
  • Client-side data exposure (IndexedDB, localStorage)
  • Dependency vulnerabilities with real-world exploit potential
  • Logic bugs in cryptographic or encoding/decoding tools

Out of Scope

  • Denial-of-service (DoS) attacks — all processing is client-side
  • Social engineering attacks
  • Issues requiring physical access to a user's device
  • Vulnerabilities in third-party services (Vercel infrastructure)
  • Self-XSS that cannot affect other users

How to Report

Send reports to: security@kaystons.dev

Please include:

  • Description of the vulnerability and its impact
  • Steps to reproduce (proof-of-concept if possible)
  • Affected browser, OS, and version
  • Any suggested mitigations

Safe Harbor

We will not pursue legal action against researchers who: act in good faith, avoid privacy violations and service disruption, do not access or modify user data, and report findings promptly.

Acknowledgments

We thank all security researchers who responsibly disclose vulnerabilities to us.
